

# 项目目标

做一把：

```text
60% 真分体 Alice
极致减薄
无线
满键 RGB
中置 Force Touch 触摸板
8 脚撑姿态系统
合体蛇形中缝
保留现有实战手感
```

核心原则：

```text
现有键盘 = 人体工学母版
不重新发明布局
不强行填满空白区
手掌基本不移动
只移动手指
空白区继续服务掌托/结构/触摸板
```

```text
35mm 触摸板放在左 B 区域附近不会误触
不需要改变左右半区夹角
不破坏原来的手感
```

这点很重要。
所以后续 CAD 里**不能再为了触摸板扭布局**，只按这个方案锁死。

# 触摸板方案

由“双侧边触摸板”改为：

```text
单个中置触摸板
位置：左 B 删除后腾出的中区
尺寸：35mm 级
主要由左手控制
```

首选硬件：

```text
Cirque TM035035 35mm 圆形轨迹板
```

备选：

```text
Cirque 23mm / 16mm，如果能买到
Azoteq 小尺寸 trackpad
CST816S 类坐标触摸模块，仅作为低价实验
```

# RGB 定稿

必须满血 RGB。

RGB 方案：

```text
每个轴位一颗 SK6812 MINI-E / 同类反贴 RGB
板载满键 RGB
嘉立创 SMT 贴装
每半边独立灯链
额外预留外接灯条接口
```

轴体是下灯位、有导光柱，所以：

```text
RGB 采用下灯位
灯珠对准轴体导光柱
不直射眼睛
通过轴体导光柱和键帽扩散
```

因为 PCB 是 Alice 扇形排布，所以规则是：

```text
每个轴位使用自身局部坐标
轴体旋转多少，RGB 跟着旋转多少
热插拔轴座也跟着旋转
灯永远贴着轴体导光柱
```

PCB 上要做：

```text
每半边 RGB 独立供电
RGB 电源 MOS / 负载开关
RGB 大电容焊盘
RGB 数据电阻
必要时加电平转换
外接灯条接口
```

外接灯条接口建议：

```text
VLED
GND
DATA_OUT
备用 GPIO / DATA
```

壳体预留：

```text
蛇形中缝导光槽
底边导光槽
触摸板周围导光边
灯条线槽
```

# PCB 路线

优先做：

```text
2 层 PCB
单面 SMT
```

画不出来再上 4 层。

原因：

```text
nRF52840 是模块，不是裸芯片射频设计
键盘矩阵低速
RGB 是串联数据
触摸板 SPI/I2C 线不多
DRV2605L 也是 I2C
```

所以 4 层不是一开始的刚需。

成本/工艺控制重点：

```text
尽量把 SMT 元件放同一面
争取嘉立创单面贴
避免双面 SMT 工程费暴涨
```

建议 SMT 面放：

```text
热插拔轴座
贴片二极管
SK6812 MINI-E RGB
RGB 去耦电容
RGB 数据电阻
DRV2605L
DRV2605L 周边阻容
FFC 座
JST 座
MOS / 负载开关
小电阻小电容
```

后装/手焊：

```text
nRF52840 主控
触摸板
FSR
LRA 马达
电池
旋钮
磁铁
弹片
外接灯条
```

# 触觉反馈

每半边预留：

```text
DRV2605L
LRA 线性马达 / iPhone Taptic 拆机马达
```

用途：

```text
触摸板 Force Touch 点击反馈
层切换震动
蓝牙切换震动
电量提示震动
后期可做按键随动震动
```

电气要求：

```text
每半边本地马达本地供电
不要让马达瞬态电流跨中缝触点
不要拉崩主控供电
```

# 电源路线

左右各一块软包锂电池，带保护板。

电池放：

```text
底壳中央腹部空腔
```

电池保护：

```text
电池上方加连续硬隔片
FR4 / PC / PET 都可
```

电源分区：

```text
主控 / 触摸板 / FSR / DRV2605L 一路
RGB 单独一路
RGB 可被 MOS 彻底断电
```

# 合体 / 分体结构

主力形态：

```text
真分体
左右完全拉开
小臂自然展开
```

便携形态：

```text
合体
左右半岛咬合
蛇形阶梯闪电缝
```

合体中缝：

```text
沿键帽错落走
围绕 6 / 7 内侧边缘避让
最小不刮擦间隙 ≥ 1.2mm
Alice 几何自然形成上窄下宽
```

合体结构：

```text
磁铁定位
榫舌 / Lip 台阶
金手指 / 弹片触点
```

触点用途：

```text
合体检测
充电 / 维护
可选辅助连接
不承担主力分体通信
```

主力通信走 BLE/ZMK。

# 脚撑系统

左右各 4 个，共 8 个。

但定义不是 8 个同时展开，而是：

```text
每半边 4 个脚撑 = 姿态组合库
```

每半边：

```text
内侧上/下脚撑：负责 tenting 帐篷角
外侧上/下脚撑：负责 tilt 前后倾角
```

实际使用：

```text
按姿态组合展开
不是四个硬点同时落地
```

外壳底部：

```text
外侧大倒角
外沿贴斜面防滑脚垫
脚撑端贴发泡橡胶垫
```

发泡垫作用：

```text
吃制造误差
吃桌面不平
不是强行解决 8 点同时支撑
```
